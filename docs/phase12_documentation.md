# Phase 12: Streamlit Dashboard — Documentation

## Overview

Phase 12 implements an interactive Streamlit web dashboard (`dashboard/app.py`) that
allows users to browse, visualize, and compare experiment results generated by the
local and Kubernetes runners.

## What Was Done

### File Created

| File                | Purpose                               |
|---------------------|---------------------------------------|
| `dashboard/app.py`  | Streamlit application (single-file)   |

### Dashboard Features

1. **Sidebar — File Selector**
   - Scans `logs/` directory for `*.json` result files (newest first).
   - Auto-detects file mode: *local*, *kubernetes*, or *combined*.
   - Optional second-file loader for cross-experiment comparison.
   - Display-mode radio: Both / Local only / K8s only.
   - Optional auto-refresh toggle with configurable interval.

2. **Summary Metrics Panel**
   - When both modes are present: comparison table (Metric / Local / K8s / Ratio).
   - When only one mode is present: four KPI metric cards (latency, throughput,
     power, energy).

3. **GPU Power Consumption Over Time**
   - Line chart from `gpu_samples[].power_w`.
   - Overlays local and per-K8s-node traces.

4. **GPU Utilization Over Time**
   - Line chart from `gpu_samples[].gpu_utilization_pct`.

5. **Latency Comparison**
   - Bar chart of average latency (local vs K8s).
   - Line chart of per-iteration latency distribution.

6. **Energy Comparison**
   - Bar chart of total energy consumed.
   - Bar chart of energy per inference.

7. **Per-Chunk Latency (K8s)**
   - Bar chart of `per_chunk_avg_latency_ms` with network overhead caption.

8. **Experiment Configuration** (collapsible expander)
   - Displays `experiment_config` JSON from each mode.

9. **Raw JSON Viewer** (collapsible expander)
   - Full raw JSON of the loaded result file.

### Data Flow

```
logs/*.json  ──▶  _load_json() (cached)
                       │
                       ▼
               _detect_mode()  ──▶  local_data / k8s_data
                       │
                       ▼
              Dashboard panels (Streamlit widgets + Pandas DataFrames)
```

## How It Was Done

### Technology

- **Streamlit 1.55** — page layout (`st.set_page_config(layout="wide")`), sidebar
  widgets, `st.line_chart`, `st.bar_chart`, `st.dataframe`, `st.metric`, `st.json`.
- **Pandas** — DataFrame construction for chart data and summary table.
- **`@st.cache_data`** — caches JSON reads to avoid re-parsing on every rerun.

### Key Design Decisions

| Decision                       | Rationale                                      |
|--------------------------------|------------------------------------------------|
| Single-file app                | Minimal complexity; all panels in one page      |
| No dependency on `analyzer.py` | Dashboard reads raw JSON directly so it can run without importing project modules |
| Auto-detect mode               | Supports local-only, k8s-only, and combined files without manual mode selection |
| `KAI_LOGS_DIR` env var         | Allows overriding the default `logs/` directory |
| String formatting in table     | Avoids pyarrow type-mixing errors in DataFrames |

### Testing & Verification

1. Installed Streamlit 1.55 and pandas via `pip install streamlit pandas`.
2. Generated a sample combined result file in `logs/` with mock GPU samples,
   latencies, power, and energy data.
3. Ran `python -m streamlit run dashboard/app.py --server.headless true` — app
   launched successfully on `http://localhost:8501` with no errors.
4. All panels rendered: summary table, power chart, utilization chart, latency
   comparison, energy comparison, per-chunk latency, config expander, raw JSON.

### Running the Dashboard

```bash
# Standard launch
python -m streamlit run dashboard/app.py

# With custom logs directory
KAI_LOGS_DIR=path/to/results python -m streamlit run dashboard/app.py

# Headless (e.g. on a server)
python -m streamlit run dashboard/app.py --server.headless true --server.port 8501
```
